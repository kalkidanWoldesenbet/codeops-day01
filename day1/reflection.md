# Reflection

## 1. What is the difference between Git and GitHub?

Git is a version control system used to track changes in files and manage project history on a local machine. GitHub is an online platform that hosts Git repositories, making it easier to store projects remotely, collaborate with others, and share code.

## 2. What is the purpose of the terminal?

The terminal is a command-line interface that allows users to interact with the computer by typing commands. It is used to create files and folders, move around directories, run programs, and execute Git commands efficiently.

## 3. What does `git add` do?

The `git add` command stages changes so they are ready to be included in the next commit. It tells Git which modified or new files should be saved in the project history.

## 4. What is the purpose of `git commit`?

The `git commit` command saves the staged changes into Git history as a snapshot of the project at a specific point in time. It helps track progress and makes it possible to review or restore earlier versions.

## 5. Write the daily Git workflow in the correct order.

1. Check the repository status with `git status`
2. Make changes to files
3. Stage changes with `git add`
4. Commit changes with `git commit -m "message"`
5. Push changes to GitHub with `git push`
